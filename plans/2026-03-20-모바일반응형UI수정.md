# 모바일 반응형 UI 수정 계획

## Context

현재 앱이 데스크톱/태블릿 기준으로 제작되어 모바일(320~430px)에서 글자 겹침, 레이아웃 깨짐이 발생함.
별도 모바일 버전을 만들지 않고, **기존 코드에 Tailwind 반응형 클래스를 추가**하여 해결한다.

## 현재 문제 요약

| 심각도 | 문제 | 파일 |
|--------|------|------|
| 🔴 CRITICAL | 대회 추가 폼 고정 너비(w-48, w-40, w-28)가 모바일에서 넘침 | TournamentList.jsx |
| 🔴 CRITICAL | 점수 입력 테이블 13+컬럼, 버튼 4개가 한 줄에 배치 | ScoreTable.jsx |
| 🔴 CRITICAL | 전체현황 테이블 min-w 고정값으로 넘침 | OverviewTab.jsx |
| 🟠 HIGH | 개인전/장려상 좌우 분할 테이블이 모바일에서 깨짐 | IndividualTab.jsx, EncouragementTab.jsx |
| 🟠 HIGH | 단체전 테이블 min-w-[120px] 고정 | TeamTab.jsx |
| 🟡 MEDIUM | 제목 text-3xl 고정, 버튼 text-lg 고정 | 전체 컴포넌트 |
| 🟡 MEDIUM | px-6 패딩이 모바일에서 과도함 | SummaryPage.jsx |

## 수정 전략

**원칙: 데스크톱 레이아웃은 유지하면서 모바일에서만 조정**

Tailwind의 반응형 접두사(`sm:`, `md:`)를 활용하여 모바일-퍼스트로 전환.

## 수정 파일 및 내용

### 1. TournamentList.jsx
**경로:** `src/components/tournament/TournamentList.jsx`

- 헤더: `text-3xl` → `text-xl sm:text-2xl md:text-3xl`
- 대회 추가 폼: `flex items-end gap-4` → `flex flex-col md:flex-row md:items-end gap-2 md:gap-4`
- 고정 너비 제거: `w-48`, `w-40`, `w-28` → `w-full md:w-48`, `w-full md:w-40`, `w-full md:w-28`
- 버튼: `px-5 py-2.5` → `px-3 py-2 sm:px-5 sm:py-2.5`, `text-sm`으로 축소
- 버튼 영역: `flex gap-3` → `flex gap-2 sm:gap-3`

### 2. ScoreTable.jsx
**경로:** `src/components/score/ScoreTable.jsx`

- 상단 버튼들: 4개 버튼 한 줄 → `flex flex-wrap gap-2`로 변경, `text-lg` → `text-sm sm:text-lg`
- 테이블 컨테이너: 모바일 스크롤 힌트 추가 (그라데이션 또는 안내 텍스트)
- 헤더 텍스트: 모바일에서 약어 사용 고려 (예: "36홀 합계" → 그대로 두되 `whitespace-nowrap` 추가)
- 패딩: `px-4 py-3` → `px-2 py-2 sm:px-4 sm:py-3`
- 제목: `text-2xl` → `text-lg sm:text-2xl`

### 3. SummaryPage.jsx
**경로:** `src/components/summary/SummaryPage.jsx`

- 패딩: `px-6 py-4` → `px-2 py-2 sm:px-4 sm:py-3 md:px-6 md:py-4`
- 탭 버튼: 텍스트 크기 `text-sm sm:text-base`
- 제목: 반응형 크기 적용

### 4. OverviewTab.jsx
**경로:** `src/components/summary/tabs/OverviewTab.jsx`

- 테이블 `text-sm` → `text-xs sm:text-sm`
- 셀 패딩 축소: `px-3 py-2` → `px-1 py-1 sm:px-3 sm:py-2`
- min-w 값들 축소 또는 제거
- `whitespace-nowrap` 추가하여 텍스트 줄바꿈 방지

### 5. IndividualTab.jsx / EncouragementTab.jsx
**경로:** `src/components/summary/tabs/IndividualTab.jsx`, `EncouragementTab.jsx`

- `min-w-[80px]`, `min-w-[60px]` → `min-w-[40px] sm:min-w-[60px] md:min-w-[80px]`
- 테이블 텍스트: `text-xs sm:text-sm`
- 셀 패딩 축소

### 6. TeamTab.jsx
**경로:** `src/components/summary/tabs/TeamTab.jsx`

- `min-w-[120px]` → `min-w-[60px] sm:min-w-[90px] md:min-w-[120px]`
- 테이블 텍스트: `text-xs sm:text-sm`

### 7. ClubManagement.jsx / ClubMemberList.jsx
**경로:** `src/components/club/ClubManagement.jsx`, `ClubMemberList.jsx`

- 헤더: `text-3xl` → `text-xl sm:text-2xl md:text-3xl`
- 패딩: `p-6` → `p-3 sm:p-4 md:p-6`
- 폼 입력: 고정 너비를 반응형으로 전환

### 8. App.jsx
**경로:** `src/App.jsx`

- 풋터 텍스트: `text-sm` → `text-xs sm:text-sm`
- 풋터 gap: `gap-4` → `gap-2 sm:gap-4`

## 작업 순서

1. TournamentList.jsx (대회 목록 - 진입점)
2. ScoreTable.jsx (점수 입력 - 가장 복잡)
3. SummaryPage.jsx + 4개 탭 컴포넌트
4. ClubManagement.jsx + ClubMemberList.jsx
5. App.jsx (풋터)

## 검증 방법

1. `npm run dev`로 개발 서버 실행
2. 브라우저 DevTools에서 모바일 뷰포트로 확인:
   - iPhone SE (375px)
   - iPhone 14 (390px)
   - Galaxy S21 (360px)
3. 각 화면별 확인 사항:
   - 글자 겹침 없음
   - 가로 스크롤 최소화 (테이블 제외)
   - 버튼/폼 터치 가능한 크기 유지
   - 데스크톱에서 기존 레이아웃 변화 없음
