# PDF 표 크기 일정화

## 날짜: 2026-03-22

## 문제
- 전체현황 탭에서 24명씩 페이지를 분할하여 PDF를 생성함
- 마지막 페이지에 24명 미만이 남으면 캔버스 높이가 작아짐
- `addCanvasToPage`가 항상 페이지 폭에 맞춰 스케일링하므로, 행이 적은 페이지는 표가 확대되어 보임
- 페이지마다 표 크기가 달라져 사용자 경험이 좋지 않음

## 해결 방안
- 전체현황 페이지 분할 시, 첫 번째 페이지(24행)의 캔버스 비율을 기준 스케일로 저장
- 이후 페이지(마지막 페이지 포함)도 동일한 스케일(imgWidth)을 적용
- 행이 적은 페이지는 높이만 줄어들고, 폭과 글자 크기는 동일하게 유지됨

## 수정 파일
- `src/hooks/usePdfDownload.js`: 전체현황 페이지 분할 로직에 고정 스케일 적용

## 구현 상세
1. 첫 페이지 캔버스의 `width/height` 비율로 기준 `imgWidth` 계산
2. 이후 페이지는 기준 `imgWidth`를 사용하여 이미지 크기 결정
3. 기존 `addCanvasToPage` 함수는 다른 섹션에서 그대로 사용
