# 로컬 데이터 → Supabase 마이그레이션 + 필수 로그인

## Context

현재 대회/클럽/회원/소속 데이터가 localStorage와 IndexedDB에 저장되어 기기 간 동기화가 불가능함.
이를 Supabase DB로 이전하고, 앱 사용 시 로그인을 필수로 요구하여 어느 기기에서든 동일한 데이터를 조회할 수 있도록 함.

---

## Step 1: Supabase 테이블 생성 (마이그레이션 SQL)

`mcp__supabase__apply_migration` 으로 아래 4개 테이블 + RLS + 마이그레이션 RPC 생성

### user_tournaments
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | BIGINT PK | Date.now() (기존 패턴 유지) |
| user_id | UUID FK → auth.users | 소유자 |
| name | TEXT | 대회명 |
| date | VARCHAR(10) | YYYY-MM-DD |
| hole_count | SMALLINT | 18/27/36 |
| group_count | SMALLINT | 조 수 |
| club_type | VARCHAR(20) | 'club' / 'affiliation' |
| players | JSONB | 선수 배열 (기존 구조 그대로) |
| created_at | BIGINT | 생성 타임스탬프 |

### user_clubs
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | BIGSERIAL PK | 자동 증가 |
| user_id | UUID FK → auth.users | 소유자 |
| name | TEXT | 클럽명 (user_id + name UNIQUE) |

### user_affiliations
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | BIGSERIAL PK | 자동 증가 |
| user_id | UUID FK → auth.users | 소유자 |
| name | TEXT | 소속명 (user_id + name UNIQUE) |

### user_members
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | BIGSERIAL PK | 자동 증가 |
| user_id | UUID FK → auth.users | 소유자 |
| name | TEXT | 이름 |
| gender | TEXT | 성별 |
| club | TEXT | 소속 클럽명 |
| birth_date | TEXT | 생년월일 (선택) |

**모든 테이블에 RLS 활성화** → `auth.uid() = user_id` 조건으로 SELECT/INSERT/UPDATE/DELETE 정책 적용

### 마이그레이션 RPC 함수
`migrate_local_data(p_tournaments, p_clubs, p_affiliations, p_members)` — 로컬 데이터를 한 번에 bulk insert (`ON CONFLICT DO NOTHING`으로 멱등성 보장)

## Step 2: Supabase 데이터 레이어 (`src/utils/supabaseData.js` 신규)

localStorage/IndexedDB 대체 함수들:

```
// 대회
loadTournaments(userId) → tournament[]
saveTournament(tournament) → upsert
deleteTournament(tournamentId) → delete

// 클럽
loadClubs() → club[]
addClub(name) → club
updateClub(id, newName) → void
deleteClub(id) → void

// 소속
loadAffiliations() → affiliation[]
addAffiliation(name) → affiliation
updateAffiliation(id, newName) → void
deleteAffiliation(id) → void

// 회원
loadMembers() → member[]
addMember(name, gender, club, birthDate) → member
updateMember(id, updates) → void
deleteMember(id) → void
updateMembersClub(oldClub, newClub) → void (클럽명 변경 시 일괄 업데이트)
```

RLS가 user_id 필터링을 처리하므로 쿼리는 단순함.

## Step 4: 로컬 데이터 → Supabase 마이그레이션 훅 (`src/hooks/useDataMigration.js` 신규)

첫 로그인 시 로컬 데이터가 있으면 Supabase로 이전:

1. `localStorage.getItem('parkgolf-migration-done-{userId}')` 체크
2. localStorage에서 tournaments, IndexedDB에서 clubs/members/affiliations 읽기
3. 데이터가 있으면 `migrate_local_data` RPC 호출
4. 성공 시 마이그레이션 완료 플래그 설정
5. 마이그레이션 중 로딩 오버레이 표시

## Step 5: 훅 수정 (localStorage/IndexedDB → Supabase)

### [useTournaments.js](src/hooks/useTournaments.js)
- `useAuthContext()`에서 `user.id` 가져오기
- `loadTournaments()` → `supabaseData.loadTournaments()`
- `saveTournaments()` useEffect 제거 → 각 mutation에서 직접 Supabase 호출
- `updatePlayer` 시 **500ms 디바운스**로 Supabase 저장 (빠른 점수 입력 대응)
- React state는 즉시 업데이트 (UI 반응성 유지)

### [useClubs.js](src/hooks/useClubs.js)
- IndexedDB 호출 → `supabaseData` 함수로 교체
- `clubs: clubs.map(c => c.name)` 인터페이스 유지

### [useMembers.js](src/hooks/useMembers.js)
- IndexedDB 호출 → `supabaseData` 함수로 교체

### [useAffiliations.js](src/hooks/useAffiliations.js)
- IndexedDB 호출 → `supabaseData` 함수로 교체

### 기본 데이터 시딩
신규 가입 + 로컬 데이터 없는 경우 → 기본 클럽(방배, 양재 등 13개), 기본 소속(서초, 강남, 송파) 자동 생성

## 설계 결정사항

1. **Players를 JSONB로 유지** — 기존 코드가 `tournament.players` 배열을 직접 참조하므로 별도 테이블보다 JSONB가 변경 범위 최소화
2. **localStorage 캐시 없이 Supabase 직접 사용** — 크로스 디바이스 동기화가 목표이므로 로컬 캐시 불필요. React state가 세션 내 캐시 역할
3. **점수 입력 디바운스 500ms** — 빠른 점수 입력 시 과도한 API 호출 방지

## 검증 방법

1. 로그인 안 한 상태에서 앱 접속 → 로그인 화면만 표시되는지 확인
2. 로그인 후 기존 로컬 데이터가 Supabase로 마이그레이션되는지 확인
3. 대회 생성/수정/삭제가 Supabase에 반영되는지 확인
4. 클럽/소속/회원 CRUD가 정상 동작하는지 확인
5. 다른 브라우저에서 같은 계정 로그인 → 동일 데이터 조회 가능한지 확인
6. 점수 입력 후 새로고침 → 데이터 유지 확인
7. Supabase 대시보드에서 RLS 정책이 다른 유저 데이터 차단하는지 확인
