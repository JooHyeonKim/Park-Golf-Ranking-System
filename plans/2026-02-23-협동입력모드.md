# 협동입력모드 구현 플랜

## Context

현재 앱은 완전 오프라인 PWA로, 한 기기에서 모든 점수를 입력하는 방식(혼자입력모드)만 지원한다.
대회 현장에서는 여러 명이 동시에 각 조의 점수를 입력하고, 2인 교차검증을 통해 기록의 정확성을 보장해야 한다.
이를 위해 Firebase Firestore 기반의 실시간 협동입력모드를 추가한다.

---

## 1. 전체 화면 흐름

```
앱 시작 → [모드 선택 화면]
  ├── [혼자입력] → TournamentList → 조 선택 → 스코어카드(탭) → 결과 보기
  │     ↑ ← ← ← ← [되돌아가기 가능]
  └── [협동입력] → 역할 선택
        ↑ ← ← ← ← [되돌아가기 가능]
        ├── [팀장] → 대회 생성 → 대회코드 발급 → 사전정보 등록 → 실시간 대시보드
        └── [참여자] → 코드 입력 → 대회 진입 → 조 선택 → 스코어카드(탭) → 제출/검증
```

**핵심 변경: 모든 화면에서 모드 선택 화면으로 되돌아가기 가능**
- 혼자입력: TournamentList 헤더에 `← 모드 선택` 뒤로가기 버튼 추가
- 협동입력: 이미 CollabRoleSelect에 `onBack` 구현되어 있음 ✓

## 3. Firestore 데이터 모델

### `tournaments/{docId}` 문서
```js
{
  code: string,              // 6자리 영숫자 (예: "HK3N7P"), unique
  name: string,              // 대회명
  date: string,              // "YYYY-MM-DD"
  holeCount: 18 | 36,
  status: 'setup' | 'active' | 'completed',
  leaderDeviceId: string,    // 팀장 기기 UUID
  createdAt: Timestamp,

  // 홀별 파 (각 9개 정수 배열)
  parA: number[],            // [4, 3, 5, 4, 3, 4, 5, 3, 4]
  parB: number[],
  parC: number[] | null,     // 18홀이면 null
  parD: number[] | null,

  // 이 대회에 참여하는 클럽 목록
  clubs: string[],
}
```

### `tournaments/{docId}/groups/{groupNumber}` 서브컬렉션
조 구조는 기존 ScoreTable과 동일. 기본 4명, "A-1-1" 형태로 추가 가능.

```js
{
  groupNumber: number,       // 1~36
  course: string,            // "A-1", "A-2", ..., "D-9"
  players: [
    { slot: 0, name: string, gender: string, club: string, course: string },
    { slot: 1, ... },
    // ...최대 8명 (기본 4 + 추가 4)
  ],

  // 제출 데이터 (기기ID를 키로)
  submissions: {
    [deviceId: string]: {
      nickname: string,
      submittedAt: Timestamp,
      scores: {
        [slotIndex: string]: {
          A: number[],       // A1~A9 (9개 점수)
          B: number[],
          C?: number[],      // 36홀만
          D?: number[],
        }
      }
    }
  },

  // 검증 상태
  verificationStatus: 'pending' | 'verified' | 'conflict',
  verifiedScores: { ... } | null,
  discrepancies: [
    { playerSlot: number, course: string, hole: number, values: { [deviceId]: number } }
  ] | null,
}
```

## 6. 팀장 플로우 상세

### Step 1: 대회 생성
- 대회명, 날짜, 홀 수(18/36) 입력
- 생성 시 6자리 고유 코드 발급 (Firestore에서 중복 체크)
- 코드는 복사/공유 가능하도록 UI 제공

### Step 2: 파 설정
- 코스별 탭 (A, B, C, D)
- 각 코스에 9개 홀의 파 값 입력 (기본값 4)
- 코스 합계 파 자동 계산 표시

### Step 3: 클럽/선수 등록 및 조 편성
- **기존 ClubManagement/ClubMemberList UI 패턴을 재사용**하되 데이터는 Firestore에 저장
- 기존 `useClubs`, `useMembers` 훅 참고: `src/hooks/useClubs.js`, `src/hooks/useMembers.js`
- 조 편성은 기존 ScoreTable.jsx의 조 구조와 동일:
  - `src/utils/data.js`의 `createInitialPlayers()` 로직 재사용
  - 36홀: 36개 조 (A-1 ~ D-9), 각 조 기본 4명
  - 18홀: 18개 조 (A-1 ~ B-9), 각 조 기본 4명
  - 추가 선수: "A-1-1" 형태 (기존 `addPlayerToCourse()` 로직 재사용)
- 각 조에 선수 이름/성별/클럽 배정
- 기존 IndexedDB의 회원 데이터로 자동완성 (기존 `searchByName()` 재사용)

### Step 4: 대회 활성화 → 실시간 대시보드
- status를 'active'로 변경하면 참여자가 입력 가능
- 대시보드에서 전체 조의 상태를 실시간 모니터링:
  - 미제출 (0/2) → 회색
  - 1명 제출 (1/2) → 노란색
  - 검증 완료 (2/2 일치) → 초록색
  - 충돌 (불일치) → 빨간색
- 모든 조가 검증 완료되면 "결과 보기" 버튼 활성화

### 결과 변환
- `getVerifiedResults()` 함수로 Firestore 데이터를 기존 tournament 객체 형태로 변환
- 변환 대상: `src/utils/data.js`의 player 구조 (id, group, course, name, gender, club, scoreA~D, detailScores)
- 변환된 객체를 기존 `SummaryPage.jsx`에 그대로 전달 → 기존 랭킹/탭 UI 재사용

## 7. 참여자 플로우 상세

### Step 1: 코드 입력
- 6자리 코드 + 닉네임 입력
- Firestore에서 코드로 대회 검색 (`where('code', '==', inputCode)`)
- 매칭 실패 → "대회를 찾을 수 없습니다" 에러
- 매칭 성공 → 대회명 표시, 확인 후 진입

### Step 2: 조 선택
- 전체 조 목록을 그리드로 표시
- 각 조의 제출 상태 표시 (0/2, 1/2, 검증완료, 충돌)
- 이미 본인이 제출한 조는 별도 표시
- 조 클릭 → 해당 조의 선수 목록 확인 후 스코어카드 진입

### Step 3: 스코어카드 입력 (핵심 UI)
- **모바일 최적화 UI** (아래 섹션 8 참조)
- 해당 조의 모든 선수에 대해 A1~D9 홀별 점수 입력
- A, B, C, D 코스 합계는 자동 계산
- 모든 홀이 채워져야 제출 버튼 활성화

### Step 4: 제출 후 상태
- 제출 완료 → Firestore의 group 문서에 submission 추가
- 대기 중 (1/2): "다른 참여자의 입력을 기다리는 중..."
- 일치 (2/2): "검증 완료! 기록이 승인되었습니다."
- 충돌: 불일치 홀 목록 표시 + "다시 입력하기" 버튼
  - 충돌 시 두 제출 모두 초기화, 두 참여자 모두 재입력

## 8. 스코어카드 탭 UI 설계 (혼자입력 + 협동입력 공용)

**참고 이미지: 실제 파크골프 대회 집계표 양식**

이 UI는 혼자입력(ScoreTable)과 협동입력(CollabScoreCard) 양쪽 모두 적용한다.
실제 종이 스코어카드와 동일한 레이아웃으로, 3개 탭으로 구성된다.

---

### 8.1 전체 레이아웃

```
┌──────────────────────────────────────────────────┐
│ ← 뒤로   대회명 - 날짜         [등수계산] [결과]  │ 상단 헤더
├──────────────────────────────────────────────────┤
│  ◀  [1조 (A-1)]  ▶                               │ 조 선택 네비게이션
├──────────────────────────────────────────────────┤
│  [집계표]  [A코스]  [B코스]  [C코스]  [D코스]     │ 탭 바
├──────────────────────────────────────────────────┤
│                                                  │
│  (선택된 탭의 콘텐츠)                              │
│                                                  │
└──────────────────────────────────────────────────┘
```

- **상단 헤더**: 뒤로가기, 대회명/날짜, 등수 계산, 결과 보기
- **조 선택**: ◀ ▶ 화살표로 조 이동 (1조~36조), 혹은 드롭다운으로 직접 선택
- **탭 바**: 집계표 / A코스 / B코스 (36홀이면 C코스 / D코스 추가)

---

### 8.2 집계표 탭 (종이 양식 좌측과 동일)

종이 스코어카드의 "집계표" 영역을 그대로 재현한다.
행: 코스별 점수 + 소계/계, 열: 선수별 (기본 4명)

```
┌──────────────────────────────────────────────────┐
│                    집 계 표                        │
├──────┬──────┬────────┬────────┬────────┬────────┤
│      │ 코스 │ 선수1  │ 선수2  │ 선수3  │ 선수4  │
│      │      │ 이학림 │ 구현정 │ 심상선 │ 구자명 │
│      │      │(남/방배)│(여/양재)│(남/서초)│(남/반포)│
├──────┼──────┼────────┼────────┼────────┼────────┤
│      │  A   │   42   │   43   │   50   │   51   │
│  전  │  B   │   33   │   37   │   40   │   37   │
│      │ 소계 │   75   │   80   │   90   │   88   │
├──────┼──────┼────────┼────────┼────────┼────────┤
│      │  C   │   --   │   --   │   --   │   --   │
│  후  │  D   │   --   │   --   │   --   │   --   │
│      │ 소계 │   --   │   --   │   --   │   --   │
├──────┴──────┼────────┼────────┼────────┼────────┤
│     계      │   --   │   --   │   --   │   --   │
├─────────────┼────────┼────────┼────────┼────────┤
│    순위     │   --   │   --   │   --   │   --   │
└─────────────┴────────┴────────┴────────┴────────┘
```

**집계표 특징:**
- **전(전반)**: A코스 + B코스, 소계 = A + B
- **후(후반)**: C코스 + D코스, 소계 = C + D (36홀만)
- **계**: A + B + C + D 합계
- **순위**: 등수 계산 후 표시 (미계산 시 `-`)
- 18홀 대회: 후반(C/D) 행 숨김, 소계 = 계
- 각 선수 헤더: 이름, (성별/클럽) 표시
- 점수 셀 클릭 → 해당 코스 탭으로 이동

---

### 8.3 A코스 탭 (종이 양식 중앙과 동일)

종이 스코어카드의 "A(Red)코스" 영역을 그대로 재현한다.
행: 9개 홀, 열: 파 + 선수별 점수

```
┌──────────────────────────────────────────────────┐
│                A(Red) 코스                         │
├──────┬──────┬────────┬────────┬────────┬────────┤
│  홀  │  파  │ 이학림 │ 구현정 │ 심상선 │ 구자명 │
├──────┼──────┼────────┼────────┼────────┼────────┤
│  1   │  3   │  [3]   │  [3]   │  [6]   │  [5]   │
│  2   │  3   │  [3]   │  [5]   │  [6]   │  [3]   │
│  3   │  6   │  [6]   │  [4]   │  [8]   │  [4]   │
│  4   │  4   │  [4]   │  [8]   │  [6]   │  [8]   │
│  5   │  3   │  [3]   │  [3]   │  [6]   │  [3]   │
│  6   │  3   │  [3]   │  [3]   │  [3]   │  [5]   │
│  7   │  4   │  [4]   │  [5]   │  [6]   │  [8]   │
│  8   │  4   │  [4]   │  [4]   │  [4]   │  [8]   │
│  9   │  4   │  [4]   │  [5]   │  [6]   │  [7]   │
├──────┼──────┼────────┼────────┼────────┼────────┤
│ 소계 │  34  │   42   │   43   │   50   │   51   │
└──────┴──────┴────────┴────────┴────────┴────────┘
```

**코스 탭 특징:**
- **홀 번호**: 1~9 (9개 홀)
- **파**: 대회 설정 시 입력한 각 홀의 파 값 (기본 4)
- **선수 컬럼**: 각 선수의 홀별 점수 입력 (숫자 input, 1~12)
- **소계 행**: 파 합계 + 각 선수의 9홀 합계 (자동 계산)
- **[숫자]**: 입력 필드 (type="number")
- 빈 셀: 미입력 상태
- 선수 이름은 집계표에서 입력, 코스 탭에서는 읽기전용 헤더

---

### 8.4 B코스 탭 (종이 양식 우측과 동일)

A코스 탭과 완전히 동일한 구조. B코스 9홀의 홀별 점수 입력.

```
┌──────────────────────────────────────────────────┐
│                B(Blue) 코스                        │
├──────┬──────┬────────┬────────┬────────┬────────┤
│  홀  │  파  │ 이학림 │ 구현정 │ 심상선 │ 구자명 │
├──────┼──────┼────────┼────────┼────────┼────────┤
│  1   │  4   │  [4]   │  [3]   │  [3]   │  [5]   │
│ ...  │ ...  │  ...   │  ...   │  ...   │  ...   │
│  9   │  4   │  [4]   │  [2]   │  [4]   │  [7]   │
├──────┼──────┼────────┼────────┼────────┼────────┤
│ 소계 │  36  │   33   │   37   │   40   │   39   │
└──────┴──────┴────────┴────────┴────────┴────────┘
```

C코스, D코스 탭도 동일 구조 (36홀 대회에만 표시).

---

### 8.5 데이터 연동

코스 탭에서 입력한 홀별 점수 → detailScores에 저장 → 코스 합계(scoreA/B/C/D) 자동 계산

```
코스 탭 입력 (A1~A9 홀별)
  ↓
detailScores: { A1: 3, A2: 4, ..., A9: 4 }
  ↓
scoreA = A1 + A2 + ... + A9 = 42 (자동 계산)
  ↓
집계표 탭의 A코스 행에 42 표시
  ↓
소계/계 자동 갱신
```

**기존 scoreA/B/C/D 필드와의 호환:**
- 코스 탭에서 홀별 입력 → scoreA = sum(A1~A9)으로 자동 계산하여 기존 필드에 반영
- 기존 순위 계산 로직(`useRanking`, `ranking.js`)은 scoreA/B/C/D를 사용하므로 변경 불필요

---

### 8.6 조 선택 네비게이션

```
┌──────────────────────────────────────┐
│  ◀  [1조 (A-1) ▼]  ▶               │
└──────────────────────────────────────┘
```

- **◀ ▶ 화살표**: 이전/다음 조로 이동 (1조 ↔ 36조 순환)
- **조 번호 클릭**: 드롭다운으로 전체 조 목록 표시, 직접 선택 가능
- 드롭다운에 각 조의 입력 상태 표시 (예: "1조 (A-1) - 4/4명 입력완료")
- 혼자입력 모드에서는 조 간 자유 이동
- 협동입력 모드에서는 선택된 조만 입력 가능

---

### 8.7 선수 정보 입력 (집계표 탭 내)

집계표 탭의 선수 헤더 영역에서 이름/성별/클럽 입력:

```
┌────────┐
│ [이름] │ ← 텍스트 입력 (자동완성 지원)
│ [성별] │ ← 남/여 선택
│ [클럽] │ ← 클럽 셀렉트박스
└────────┘
```

- 이름 입력 후 blur/Enter → 기존 `searchByName()` 자동완성
- 동명이인 → 기존 동명이인 선택 모달 재사용
- 클럽 → 기존 클럽 셀렉트박스 재사용

---

### 8.8 혼자입력 모드 vs 협동입력 모드 차이

| 기능 | 혼자입력 | 협동입력 |
|------|----------|----------|
| 조 선택 | ◀▶로 자유 이동 | 조 선택 화면에서 선택 후 진입 |
| 선수 정보 입력 | 집계표 탭에서 직접 입력 | 팀장이 사전 등록 (읽기전용) |
| 데이터 저장 | localStorage | Firestore |
| 등수 계산 | 전체 조 데이터 기반 | 팀장 대시보드에서 |
| 제출 버튼 | 없음 (자동 저장) | 하단에 [제출하기] 표시 |
| 미입력 표시 | 빈 셀 | 빨간 테두리 |

---

### 8.9 입력 방식 상세

- 각 홀: 숫자 input (type="number", min=1, max=12)
- 터치 친화적: 최소 44px 터치 타겟
- 입력 즉시 자동 저장 (onChange → localStorage/Firestore)
- Tab 키로 다음 셀 이동 (홀 순서대로)
- 협동입력: 미입력 홀 빨간 테두리, 입력 완료 홀 초록 배경
- 협동입력: 충돌 재입력 시 불일치 홀 노란 배경 + 이전 두 값 표시

## 9. 2인 교차검증 시스템

### 비교 로직 (`useCollabVerification.js`)
```
입력: submission1, submission2 (두 참여자의 점수 데이터)
비교: 모든 선수의 모든 홀(A1~D9) 점수를 하나씩 비교
출력: { isMatch: boolean, discrepancies: [...] }
```

### 검증 트리거
- Firestore onSnapshot으로 group 문서 감시
- `Object.keys(submissions).length >= 2` 감지 시 비교 실행
- **팀장 클라이언트**가 비교 후 결과를 group 문서에 기록
- **참여자 클라이언트**도 동일 로직 실행 (팀장 오프라인 시 대비)
- Firestore 트랜잭션으로 중복 쓰기 방지

### 충돌 처리
- 불일치 발견 → `verificationStatus: 'conflict'` + `discrepancies` 배열 기록
- 두 참여자 모두 onSnapshot으로 충돌 알림 수신
- 충돌 시 submissions 초기화 → 두 참여자 모두 재입력 필요
- 재입력 시 불일치 홀을 노란색으로 강조 표시

## 13. 검증 방법

1. **Firebase 연결 테스트**: `npm run dev` → 콘솔에서 Firestore 연결 확인
2. **팀장 플로우**: 대회 생성 → 코드 발급 → 파 설정 → 조 편성 → 대시보드 확인
3. **참여자 플로우**: 다른 브라우저/시크릿 탭에서 코드 입력 → 대회 진입 → 조 선택 → 스코어 입력 → 제출
4. **2인 검증**: 두 개 브라우저에서 같은 조에 점수 입력 → 일치 시 승인 확인 / 불일치 시 충돌 알림 확인
5. **결과 확인**: 팀장 대시보드 → "결과 보기" → 기존 SummaryPage에서 랭킹/탭 정상 표시 확인
6. **기존 기능 회귀**: 혼자입력모드가 여전히 정상 동작하는지 확인
