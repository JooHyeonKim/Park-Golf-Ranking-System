# 클럽 관리 페이지 추가

## Context

현재 클럽 목록이 `data.js`의 정적 상수(`CLUBS`)로 고정되어 있어, 클럽 이름 수정이나 새 클럽 추가가 불가능함. 메인 화면에서 진입 가능한 클럽 관리 페이지를 추가하여 클럽 목록을 동적으로 관리할 수 있도록 함. 기존 13개 클럽은 기본값으로 유지.

## 수정/생성 파일 (6개)

### 1. `src/utils/data.js` — 클럽 localStorage 함수 추가

- `CLUBS` → `DEFAULT_CLUBS`로 이름 변경 (기본값 역할)
- `loadClubs()` / `saveClubs()` 함수 추가 (기존 loadTournaments/saveTournaments 패턴 동일)
- localStorage 키: `parkgolf-clubs`
- `loadClubs()`는 저장된 데이터 없으면 `DEFAULT_CLUBS` 반환

### 2. `src/hooks/useClubs.js` — 새 파일 (커스텀 훅)

기존 `useTournaments.js` 패턴을 따르는 클럽 관리 훅:
- `clubs` 상태 + localStorage 동기화
- `addClub(name)` — 중복 검사 후 추가
- `editClub(oldName, newName)` — 중복 검사 후 수정
- `deleteClub(name)` — 삭제

### 3. `src/components/club/ClubManagement.jsx` — 새 파일 (클럽 관리 페이지)

Props: `{ clubs, onAddClub, onEditClub, onDeleteClub, onBack }`

UI 구성:
- 헤더: "← 대회 목록" 뒤로가기 + "클럽 관리" 제목
- 새 클럽 추가: 텍스트 입력 + 추가 버튼
- 클럽 목록: 각 항목에 수정/삭제 버튼
- 인라인 편집: 수정 클릭 → 입력 필드 + 저장/취소 버튼
- 중복 클럽명 에러 메시지 표시

### 4. `src/App.jsx` — 라우팅 및 연결

- `useClubs` 훅 사용
- `screenMode`에 `'clubs'` 추가
- `handleGoToClubs` 네비게이션 핸들러 추가
- **중요**: `clubs` 라우팅을 `list` 라우팅보다 앞에 배치 (`list` 조건에 `|| !currentTournament` fallback이 있으므로)
- `TournamentList`에 `onGoToClubs` prop 전달
- `ScoreTable`에 `clubs` prop 전달

### 5. `src/components/tournament/TournamentList.jsx` — 클럽 관리 버튼 추가

헤더의 기존 "+ 새 대회" 버튼 옆에 "클럽 관리" 버튼 추가 (파란색, `bg-blue-600`):
```
[클럽 관리] [+ 새 대회]
```

### 6. `src/components/score/ScoreTable.jsx` — 동적 클럽 목록 사용

- `import { CLUBS }` 제거
- `clubs` prop으로 변경
- 셀렉트 박스와 테스트 데이터에서 prop 사용

## 검증 방법

1. `npm run dev` → 메인 화면에서 "클럽 관리" 버튼 확인
2. 클럽 관리 페이지 진입 → 13개 기본 클럽 표시 확인
3. 클럽 추가/수정/삭제 동작 확인
4. 뒤로가기 → 대회 목록 복귀 확인
5. 점수 입력 화면 → 셀렉트 박스에 수정된 클럽 목록 반영 확인
6. 새로고침 후 클럽 목록 유지 확인 (localStorage 저장)
7. `npm run build` 빌드 성공 확인
