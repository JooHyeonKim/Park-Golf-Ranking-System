# 대회 생성 시 18홀/36홀 선택 기능

## Context
대회 생성 시 홀 수 구분 없이 항상 36홀(A/B/C/D 코스) 포맷으로만 생성되던 것을 개선.
18홀 대회를 지원하기 위해 생성 시 홀 수를 선택하고, 18홀이면 C코스/D코스/C+D 칼럼을 숨기는 기능 추가.

## 변경 파일 (6개)

### 1. `src/utils/data.js`
- `createTournament(name, date)` → `createTournament(name, date, holeCount)` 시그니처 변경
- 대회 객체에 `holeCount` 필드 추가 (기본값 36)

### 2. `src/hooks/useTournaments.js`
- `addTournament(name, date)` → `addTournament(name, date, holeCount)` 시그니처 변경
- `createTournament` 호출 시 `holeCount` 전달

### 3. `src/App.jsx`
- `handleAddTournament(name, date)` → `handleAddTournament(name, date, holeCount)` 시그니처 변경
- `addTournament` 호출 시 `holeCount` 전달

### 4. `src/components/tournament/TournamentList.jsx`
- `newHoleCount` 상태 추가 (기본값 36)
- 대회 생성 폼에 18홀/36홀 토글 버튼 UI 추가
- `onAdd` 호출 시 `newHoleCount` 전달
- 대회 목록 카드에 홀 수 표시 추가

### 5. `src/components/score/ScoreTable.jsx`
- `is36Hole` 변수로 홀 수 판별 (`tournament.holeCount || 36`)
- 18홀일 때: C코스, D코스, C+D 헤더/바디 칼럼 숨김
- "36홀 합계" → "18홀 합계" 텍스트 동적 변경
- 테스트 데이터 생성 시 18홀이면 scoreC/scoreD 미생성

### 6. `src/components/summary/tabs/OverviewTab.jsx`
- `is36Hole` 변수로 홀 수 판별
- ScoreTable과 동일하게 C/D/C+D 칼럼 조건부 숨김
- 합계 칼럼 텍스트 동적 변경

## 변경하지 않은 것
- 선수 구성: 18홀/36홀 모두 144명(36조) 유지
- `ranking.js`: 총합 계산이 null을 0으로 처리하므로 수정 불필요
- `useRanking.js`: 변경 불필요
- IndividualTab, EncouragementTab, TeamTab: 개별 코스 점수 미표시이므로 변경 불필요
- 기존 대회 하위 호환: `holeCount` 없으면 36으로 간주
