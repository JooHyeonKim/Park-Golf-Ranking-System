# 조 수 조정 및 전체현황 페이지네이션

## Context

### 1. 조 수 조정 기능
현재 대회 생성 시 조 수가 고정되어, 18홀=72명 / 36홀=144명이 자동 생성된다. 실제 참가 인원이 적으면 빈 칸이 점수 입력 화면과 PDF 출력에 그대로 노출된다. 사용자가 총 조 수를 지정하여 필요한 만큼만 선수 슬롯을 생성하도록 한다.

- 조 수는 총 조 수 기준 (코스당이 아님). `distributeCourseNames()`로 코스별 균등 분배
- 예: 36홀, 20조 → A:5, B:5, C:5, D:5 → 80명
- 대회 생성 시 및 점수 입력 화면에서 조 수 변경 가능
- 기존 "+ 선수 추가" 기능과 충돌 없음 (추가 선수는 기존 조 내에서 동작)

### 2. 전체현황 탭 PDF 페이지네이션
전체 PDF 다운로드(`usePdfDownload.js`)에는 전체현황을 24명씩 페이지 분할하는 기능이 있으나, 전체현황 탭의 개별 PDF 다운로드(`useSinglePdfDownload.js`)에는 없었음. 개별 PDF에도 24명씩 페이지 분할을 적용.

## 수정 파일

### 1. `src/utils/data.js` — 핵심 로직
- `distributeCourseNames(courses, groupCount)`: 총 조 수를 코스별로 균등 분배
- `createInitialPlayers(holeCount, groupCount)`: groupCount 기반 선수 슬롯 생성
- `createTournament(name, date, holeCount, groupCount)`: groupCount 필드 저장
- `adjustGroupCount(existingPlayers, holeCount, newGroupCount)`: 기존 데이터 보존하며 조 수 변경

### 2. `src/hooks/useTournaments.js` — 시그니처 전파
- `addTournament(name, date, holeCount, groupCount)`: groupCount 전달
- `updateGroupCount(tournamentId, newGroupCount)`: 점수 입력 화면에서 조 수 변경

### 3. `src/App.jsx` — 시그니처 전파
- `handleAddTournament(name, date, holeCount, groupCount)`: groupCount 전달
- `onUpdateGroupCount` prop을 ScoreTable에 전달

### 4. `src/components/tournament/TournamentList.jsx` — UI 추가
- 총 조 수 number input (1~max, max=18홀:18/36홀:36)
- 총 인원수 미리보기 텍스트 표시
- 대회 목록 카드에 조 수 병기: `36홀 / 20조`
- `getTournamentGroupCount()`: 하위 호환 (기존 groupsPerCourse → groupCount 변환)

### 5. `src/components/score/ScoreTable.jsx` — 조 수 변경 UI
- 대회 헤더에 총 조 수 변경 number input 추가
- 조 수 감소 시 데이터 유실 경고

### 6. `src/hooks/useSinglePdfDownload.js` — PDF 페이지 분할 지원
- `rowsPerPage` 매개변수 추가 (기본값 `null` = 단일 페이지)
- `rowsPerPage` 지정 시 `usePdfDownload.js`와 동일한 행 show/hide 방식으로 페이지 분할
- `captureElement()`, `addCanvasToPage()` 헬퍼 함수 추가

### 7. `src/components/summary/tabs/OverviewTab.jsx` — PDF 페이지 분할 적용
- `useSinglePdfDownload` 호출 시 4번째 인자로 `24` 전달
- 탭 UI는 변경 없음 (전체 선수 한 번에 표시)

## 하위 호환성

- 모든 매개변수에 기본값 설정 (`groupCount = null` → courses.length * 9)
- 기존 대회 데이터에 groupCount 필드가 없어도 정상 동작
- `getTournamentGroupCount()`: 기존 `groupsPerCourse` 필드도 변환 처리

## 검증

- 36홀 36조(기본) → 144명 (기존 동일)
- 36홀 20조 → 80명
- 18홀 6조 → 24명
- 기존 대회 정상 로드
- "+ 선수 추가" 정상 동작
- PDF 출력 정상 (전체 PDF + 개별 PDF 모두)
- 전체현황 탭 개별 PDF 다운로드 시 24명씩 페이지 분할 정상
- 다른 탭(개인전, 장려상, 단체전) PDF는 기존 단일 페이지 유지
