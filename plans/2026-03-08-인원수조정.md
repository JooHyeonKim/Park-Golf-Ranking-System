# 코스당 조 수 조정 기능

## Context

현재 대회 생성 시 코스당 조 수가 9로 고정되어, 18홀=72명 / 36홀=144명이 자동 생성된다. 실제 참가 인원이 적으면 빈 칸이 점수 입력 화면과 PDF 출력에 그대로 노출된다. 사용자가 코스당 조 수(1~9)를 지정하여 필요한 만큼만 선수 슬롯을 생성하도록 한다.

- 예: 36홀, 5조 → 4코스 × 5조 × 4명 = 80명
- 기존 "+ 선수 추가" 기능과 충돌 없음 (추가 선수는 기존 조 내에서 동작)

## 수정 파일 (4개)

### 1. `src/utils/data.js` — 핵심 로직
- `createInitialPlayers(holeCount, groupsPerCourse = 9)`: 16번 줄 `i <= 9` → `i <= groupsPerCourse`
- `createTournament(name, date, holeCount, groupsPerCourse)`: `groupsPerCourse` 필드 저장 및 전달

### 2. `src/hooks/useTournaments.js` — 시그니처 전파
- 47번 줄 `addTournament(name, date, holeCount = 36)` → `addTournament(name, date, holeCount = 36, groupsPerCourse = 9)`
- `createTournament` 호출 시 `groupsPerCourse` 전달

### 3. `src/App.jsx` — 시그니처 전파
- 73번 줄 `handleAddTournament(name, date, holeCount)` → `handleAddTournament(name, date, holeCount, groupsPerCourse)`
- `addTournament` 호출 시 `groupsPerCourse` 전달

### 4. `src/components/tournament/TournamentList.jsx` — UI 추가
- `newGroupsPerCourse` state 추가 (기본값 9)
- 홀 수 선택 옆에 조 수 select 드롭다운 추가 (1~9조)
- 총 인원수 미리보기 텍스트 표시
- `handleAdd`에서 `onAdd` 호출 시 4번째 인자로 전달
- 대회 목록 카드에 조 수 병기: `36홀 / 5조`

## 하위 호환성

모든 매개변수에 `groupsPerCourse = 9` 기본값 설정. 기존 대회 데이터에 필드가 없어도 정상 동작. 대회 목록 표시 시 `tournament.groupsPerCourse || 9` 사용.
